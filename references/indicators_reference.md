# Yi Stickiness 指标详解参考手册

> 本文档供 `yi-indicators` Skill 在解读原始指标数值时引用。
> 所有指标均为归一化数值，由 Yi 指标引擎（Go）基于逐笔成交数据计算。

---

## 一、指标总览

| # | 指标 | 范围 | 方向含义 | 核心测量维度 |
|---|------|------|----------|-------------|
| 1 | Liquidity | [0, 1] | 高=厚，低=薄 | 市场厚度/脆弱性 |
| 2 | Pressure | [-1, +1] | 正=买，负=卖 | 买卖压力方向 |
| 3 | Absorption | [0, 1] | 高=方向确定 | 方向+流动性复合强度 |
| 4 | TrendEff | [0, 1] | 高=趋势性强 | 趋势效率 (Kaufman ER) |
| 5 | BuyRatioOscillation | [0, 1] | 高=方向频繁切换 | 买卖方向切换频率 |
| 6 | Divergence | [-1, +1] | 正=短>长，负=短<长 | 短/长周期流动性背离 |
| 7 | TickBias | [-1, +1] | 正=涨Tick多 | 极短期逐笔偏差 |
| 8 | SizeAsymmetry | [-1, +1] | 正=买方单大 | 买卖订单规模不对称 |
| 9 | ClusterAnomaly | [0, 1] | 高=异常聚类 | 连续同侧交易异常 |
| 10 | WhaleConcentration | [0, 1] | 高=大单主导 | 成交量集中度 |

## 二、逐指标详解

### 2.1 Liquidity (流动性) — [0, 1]

**测量什么**：市场厚度，即价格对成交量的敏感程度。基于 Amihud 流动性不足度量的变体，结合自适应 K 值校准。

**数值解读**：

| 区间 | 标签 | 微结构含义 | 交易含义 |
|------|------|-----------|---------|
| [0, 0.20) | 极低 | 价格极度脆弱，少量成交即可推动价格 | 突破/崩盘风险高，适合突破策略 |
| [0.20, 0.40) | 偏低 (利区) | 价格脆弱，方向性强 | 趋势信号可靠性高，顺势操作 |
| [0.40, 0.60) | 正常偏薄 (常区) | 价格有一定缓冲 | 信号可靠性中等 |
| [0.60, 0.85) | 正常偏厚 (常区) | 价格较稳定 | 信号噪声比降低 |
| [0.85, 0.95) | 偏高 (滞区) | 市场粘滞，价格难以推动 | 能量积累中，等待方向突破 |
| [0.95, 1.00] | 极高 (滞区) | 市场极度粘滞，短期几乎推不动 | 大概率盘整，关注 Divergence 变化 |

**关键逻辑**：
- Liquidity 是**所有其他指标可靠性的前置条件**
- 低 Liquidity（利区）+ 强 Pressure → 高效的方向性行情
- 低 Liquidity（利区）+ 弱 Pressure → 可能是假突破
- 高 Liquidity（滞区）+ 任意 Pressure → 方向有效但缓慢，适合积累仓位

---

### 2.2 Pressure (压力) — [-1, +1]

**测量什么**：买方 vs 卖方的净压力方向。综合考虑成交量方向、主动成交比率、Tick 方向分布。

**数值解读**：

| 区间 | 标签 | 微结构含义 |
|------|------|-----------|
| [+0.50, +1.00] | 强买压 | 买方显著主导，主动买入远超卖出 |
| [+0.15, +0.50) | 偏买 | 买方略占优势 |
| [-0.15, +0.15) | 平衡 | 买卖力量基本均衡 |
| [-0.50, -0.15) | 偏卖 | 卖方略占优势 |
| [-1.00, -0.50] | 强卖压 | 卖方显著主导，主动卖出远超买入 |

**关键交叉验证**：
- Pressure 方向应结合 Liquidity 判断可靠性
- Pressure 绝对值很大但 TradeCount 很小 → 可能是偶发大单，不是持续趋势
- Pressure 与 buy_volume/sell_volume 比应大致一致，否则可能存在结构性偏差

---

### 2.3 Absorption (吸收度) — [0, 1]

**测量什么**：方向性确认强度。结合 Pressure 的绝对值和 Liquidity 计算，衡量"方向是否真实且有流动性支撑"。

**数值解读**：

| 区间 | 标签 | 含义 |
|------|------|------|
| [0.50, 1.00] | 高吸收 | 方向确认，流动性充足，趋势可持续 |
| [0.25, 0.50) | 中等吸收 | 有方向性但强度不够，需谨慎 |
| [0, 0.25) | 低吸收 | 缺乏方向性或流动性严重不足 |

**关键逻辑**：
- 高 Absorption = Pressure 方向 + 流动性支撑两者兼备 → 趋势可靠性最高
- 低 Absorption 但有明显 Pressure → 方向是真的但流动性不支持 → 可能短命行情
- Absorption 与 TrendEff 同时高 → 最强趋势确认信号

---

### 2.4 TrendEff (趋势效率) — [0, 1]

**测量什么**：Kaufman Efficiency Ratio 变体。衡量价格变动的方向一致性——价格是"有效率地"朝一个方向走，还是在来回波动。

**数值解读**：

| 区间 | 标签 | 含义 |
|------|------|------|
| [0.30, 1.00] | 高效率 | 价格变动方向一致，趋势性强 |
| [0.10, 0.30) | 中等效率 | 部分方向性，部分噪音 |
| [0, 0.10) | 低效率 | 噪音主导，无明确方向 |

**关键逻辑**：
- TE 高 + Pressure 方向一致 → 确认趋势方向
- TE 高 + Pressure 方向中值 → 可能有假趋势（价格在动但方向不清晰）
- TE 低 + Liquidity 高 → 典型盘整，区间交易机会
- TE 在七态分类中用于区分"劲"和"缓"（TE > 0.05 = 劲）

---

### 2.5 BuyRatioOscillation (买卖震荡) — [0, 1]

**测量什么**：买卖方向切换的频率。高频切换意味着市场犹豫，低频意味着方向稳定。

**数值解读**：

| 区间 | 标签 | 含义 |
|------|------|------|
| [0.50, 1.00] | 高震荡 | 买卖方向频繁切换，市场犹豫 |
| [0.25, 0.50) | 中震荡 | 有一定切换，但趋势偏稳 |
| [0, 0.25) | 低震荡 | 买卖方向稳定，趋势明确 |

**关键逻辑**：
- 高 BuyRatioOscillation + 低 TrendEff → 典型的无方向震荡 → 避免趋势交易
- 低 BuyRatioOscillation + 高 TrendEff → 强趋势确认
- 高 BuyRatioOscillation + 高 WhaleConcentration → 可能是大资金故意制造混乱

---

### 2.6 Divergence (背离) — [-1, +1]

**测量什么**：短期流动性（短线资金行为）与长期流动性（中长线资金行为）的差异。

**数值解读**：

| 区间 | 含义 |
|------|------|
| [+0.30, +1.00] | 短期流动性远厚于长期 → 短线资金活跃，中长线资金撤离 |
| [-0.30, +0.30) | 短长期流动性一致 → 各路资金看法一致 |
| [-1.00, -0.30] | 短期流动性远薄于长期 → 短线资金撤离，中长线资金驻守 |

**关键逻辑**：
- 正 Divergence（短>长）：短线资金活跃但中长线不跟 → 短期可能走强但持续性存疑
- 负 Divergence（短<长）：短线恐慌但中长线资金未撤离 → 短期可能超跌但底部有支撑
- Divergence 绝对值持续扩大 → 市场结构正在发生质变

---

### 2.7 TickBias (Tick偏差) — [-1, +1]

**测量什么**：极短期（Tick 级别）的涨跌偏向。如果涨 Tick 数量 > 跌 Tick 数量，TickBias 为正。

**数值解读**：

| 区间 | 含义 |
|------|------|
| [+0.30, +1.00] | 涨 Tick 居多，极短期偏多 |
| [-0.30, +0.30) | 涨跌 Tick 均衡 |
| [-1.00, -0.30] | 跌 Tick 居多，极短期偏空 |

**关键逻辑**：
- TickBias 是最高频的方向信号，变化最快
- TickBias 方向与 15m/1h Pressure 方向一致 → 多时间框架确认
- TickBias 与 Pressure 方向矛盾 → 短期可能反转
- 对 scalping 策略参考价值最高

---

### 2.8 SizeAsymmetry (大小不对称) — [-1, +1]

**测量什么**：买方平均单笔成交大小 vs 卖方平均单笔成交大小。正 = 买方平均单笔更大。

**数值解读**：

| 区间 | 含义 |
|------|------|
| [+0.20, +1.00] | 买方平均单笔更大 → 可能有聪明钱/机构买入（拆小单或大单） |
| [-0.20, +0.20) | 买卖单笔规模对称 → 市场参与者结构均衡 |
| [-1.00, -0.20] | 卖方平均单笔更大 → 可能有聪明钱/机构卖出 |

**关键逻辑**：
- SizeAsymmetry 是"聪明钱"的潜在指标——大资金通常拆单，但平均单笔仍大于散户
- 正 SizeAsymmetry + 正 Pressure → 聪明钱在买入方向
- 正 SizeAsymmetry + 负 Pressure → 可能有托盘（假买真卖）
- 需要结合 TickBias 区分拆单方向

---

### 2.9 ClusterAnomaly (聚类异常) — [0, 1]

**测量什么**：连续同侧交易（连续买入或连续卖出）的异常程度。检测算法交易/拆分大单行为。

**数值解读**：

| 区间 | 含义 |
|------|------|
| [0.50, 1.00] | 检测到显著连续同侧交易 → 疑似算法拆分单 |
| [0.20, 0.50) | 有轻微聚类特征 |
| [0, 0.20) | 成交分布正常，无聚类异常 |

**关键逻辑**：
- 高 ClusterAnomaly → 大概率有程序化交易在执行
- 结合 TickBias 方向 → 判断是程序化买入还是程序化卖出
- 高 ClusterAnomaly + 高 Absorption → 程序化在推动价格（跟庄机会）
- 高 ClusterAnomaly + 低 Absorption → 程序化被执行但未推动价格（可能是被动接单）

---

### 2.10 WhaleConcentration (大户集中度) — [0, 1]

**测量什么**：成交量在大单上的集中程度。类似 HHI 指数应用于成交量分布。

**数值解读**：

| 区间 | 含义 |
|------|------|
| [0.60, 1.00] | 高集中度 → 少数大单主导，鲸鱼活跃 |
| [0.30, 0.60) | 中等集中度 → 有一定大户参与 |
| [0, 0.30) | 低集中度 → 成交量分散，散户主导 |

**关键逻辑**：
- 高 WhaleConcentration + 方向性 Pressure → 鲸鱼在主动推动价格
- 高 WhaleConcentration + 弱 Pressure → 鲸鱼在被动挂单（做市/吸筹/派发）
- 结合 SizeAsymmetry：高 WhaleConc + 正 SizeAsym → 大资金在买；高 WhaleConc + 负 SizeAsym → 大资金在卖
- 鲸鱼活跃 + 利区 → 可能制造剧烈波动

## 三、七态分类 (Seven-State Classification)

状态由 Liquidity 和 TrendEff 两个维度交叉决定：

```
状态 = f(Liquidity, Pressure, TrendEff)

区判断:
  利区 (Liquid):   Liquidity < 0.4
  常区 (Neutral):  0.4 ≤ Liquidity ≤ 0.85
  滞区 (Stagnant): Liquidity > 0.85

方向判断:
  买方向: Pressure > 0
  卖方向: Pressure < 0

强弱判断:
  劲 (Strong): TrendEff > 0.05
  缓 (Weak):   TrendEff ≤ 0.05
```

### 3.1 利区状态 (Liquidity < 0.4)

| 状态 | 条件 | 特征 |
|------|------|------|
| 劲买 | Pressure > 0, TE > 0.05 | 低流动性 + 高效买方推动 → 强上升趋势 |
| 缓买 | Pressure > 0, TE ≤ 0.05 | 低流动性 + 弱买方推动 → 可能虚涨 |
| 劲卖 | Pressure < 0, TE > 0.05 | 低流动性 + 高效卖方推动 → 强下降趋势 |
| 缓卖 | Pressure < 0, TE ≤ 0.05 | 低流动性 + 弱卖方推动 → 可能虚跌 |

### 3.2 常区 (0.4 ≤ Liquidity ≤ 0.85)

| 状态 | 条件 | 特征 |
|------|------|------|
| 常 | 任意 Pressure | 流动性正常，市场平稳，无明确方向信号 |

### 3.3 滞区状态 (Liquidity > 0.85)

| 状态 | 条件 | 特征 |
|------|------|------|
| 滞买 | Pressure > 0 | 高流动性 + 买方试探 → 能量积累，可能向上突破 |
| 滞卖 | Pressure < 0 | 高流动性 + 卖方试探 → 能量积累，可能向下突破 |

### 3.4 状态转换价值

交易者最应关注的状态转换：
1. **滞区 → 利区**：能量释放，往往伴随强趋势 → Spring Model 核心信号
2. **利区 → 滞区**：趋势衰竭，能量重新积累 → 趋势结束信号
3. **常区 → 利区**：市场从平衡进入方向性 → 新趋势启动
4. **劲→缓**：趋势减速 → 可能在衰竭
5. **缓→劲**：趋势加速 → 可能在加速赶顶/底

## 四、多指标组合信号

### 强做多信号
```
Liquidity < 0.4 (利区)
+ Pressure > 0.15 (偏买)
+ TrendEff > 0.10 (有效率)
+ Absorption > 0.25 (中等以上吸收)
+ BuyRatioOscillation < 0.5 (方向稳定)
→ 顺势做多，可靠性高
```

### 强做空信号
```
Liquidity < 0.4 (利区)
+ Pressure < -0.15 (偏卖)
+ TrendEff > 0.10 (有效率)
+ Absorption > 0.25 (中等以上吸收)
+ BuyRatioOscillation < 0.5 (方向稳定)
→ 顺势做空，可靠性高
```

### Spring Model 做多信号（滞区转利区）
```
当前 Liquidity > 0.85 (滞区)
+ 前期 Liquidity 持续 > 0.85 (积累充分)
+ Pressure > 0 (买方试探)
+ Divergence 开始从正转负 (短线资金在启动)
+ 关注 Liquidity 跌破 0.85 → 做多入场
```

### Spring Model 做空信号（滞区转利区）
```
当前 Liquidity > 0.85 (滞区)
+ 前期 Liquidity 持续 > 0.85 (积累充分)
+ Pressure < 0 (卖方试探)
+ Divergence 开始从负转正 (短线资金在撤离)
+ 关注 Liquidity 跌破 0.85 → 做空入场
```

### 假突破预警
```
Liquidity < 0.4 (利区)
+ abs(Pressure) > 0.3 (方向明显)
+ TrendEff < 0.05 (低效率!)
+ ClusterAnomaly > 0.5 (程序化异常)
→ 可能是算法制造的假突破，谨慎追入
```

### 盘整确认
```
Liquidity > 0.6 (常区偏上)
+ abs(Pressure) < 0.15 (无明显方向)
+ TrendEff < 0.10 (低效率)
+ BuyRatioOscillation > 0.5 (方向频繁切换)
→ 明确盘整，宜区间操作，不宜追趋势
```

## 五、可用周期与用途

| 周期 | 角色 | 指标有效期 | 适用场景 |
|------|------|-----------|---------|
| 1m | 微结构确认 | 1-5 分钟 | Scalping 精确入场 |
| 3m | 微结构确认 | 3-15 分钟 | 短线入场确认 |
| 5m | 入场精调 | 5-30 分钟 | 短线交易 |
| 15m | 主要执行 | 15-90 分钟 | 日内交易核心周期 |
| 1h | 战术框架 | 1-6 小时 | 日内方向框架 |
| 4h | 结构框架 | 4-24 小时 | 波段交易 |
| 1d | 趋势定调 | 1-7 天 | 主趋势判断 |
